# handbook

Curated, opinionated notes on system design fundamentals.

Each topic answers:

- What is it?
- Why does it exist?
- How does it work?
- Trade-offs (pros, cons, when to use, when not to)
- Real-world examples
- Interview-ready summary

## Planned topics

1. **Fundamentals** — reliability, scalability, maintainability; back-of-envelope estimation; latency numbers every engineer should know
2. **Data** — SQL vs NoSQL, replication, sharding, indexing, transactions
3. **Communication** — REST vs RPC vs GraphQL, message queues, pub/sub, streaming
4. **Caching** — cache patterns, eviction policies, CDNs
5. **Consistency & consensus** — CAP, PACELC, consensus algorithms, distributed transactions
6. **Storage** — object storage, distributed file systems, time-series
7. **Search** — full-text search, indexing strategies
8. **Observability** — logs, metrics, tracing
9. **Security** — authentication, authorization, rate limiting, secrets management
